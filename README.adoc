Projecte ABP - Projecte 3 - Aplicació mobile amb API REST

== Descripció del projecte

MobiRent és una aplicació mòbil Android desenvolupada amb Kotlin i Jetpack Compose que permet consultar vehicles, veure’n el detall, consultar la disponibilitat per rang de dates i gestionar reserves mitjançant una arquitectura client-servidor basada en una API REST implementada amb Spring Boot.

L’aplicació diferencia entre funcionalitats per a usuari anònim i funcionalitats per a client autenticat. Els usuaris registrats poden iniciar sessió, editar el seu perfil, crear reserves, consultar-les, ordenar-les i anul·lar-les abans de la seva data d’inici.

== Objectiu

L’objectiu del projecte és implementar una solució completa formada per:

* Un backend REST amb Spring Boot i Maven.
* Una aplicació Android nativa amb Kotlin i Jetpack Compose.
* Persistència de dades amb MySQL.
* Comunicació desacoblada entre client i servidor seguint principis REST.
* Documentació tècnica i de seguiment del projecte dins del repositori.

== Arquitectura del sistema

El projecte segueix una arquitectura client-servidor desacoblada:

* El backend exposa una API REST.
* El client Android consumeix aquesta API mitjançant HTTP.
* La persistència es gestiona al backend.
* L’estat de sessió del client es desa al dispositiu mòbil sense guardar la contrasenya.

=== Backend

El backend està estructurat per capes:

* Capa API REST: controladors i endpoints.
* Capa de lògica de negoci: serveis.
* Capa de persistència: repositoris i entitats.

=== Mobile

L’aplicació mòbil està estructurada en capes i funcionalitats, seguint el model MVVM a la capa d’interfície:

* UI layer
* Domain layer
* Data layer

La gestió d’estat es realitza amb `MutableStateFlow` i `StateFlow`, i les operacions asíncrones es resolen amb coroutines.

== Configuració del backend

=== 1. Crear la base de dades

Crear una base de dades MySQL amb el nom:

[source,sql]
----
CREATE DATABASE [nom_base_dades];
----

=== 2. Configurar credencials

Editar el fitxer:

[source,text]
----
backend/src/main/resources/application.properties
----

i ajustar com a mínim:

[source,properties]
----
spring.datasource.url=jdbc:mysql://localhost:3306/[nom_base_dades]
spring.datasource.username=[usuari]
spring.datasource.password=[contrasenya]
spring.jpa.hibernate.ddl-auto=update
----

=== 3. Executar el backend

Des de la carpeta `backend`:

[source,bash]
----
./mvnw spring-boot:run
----

o bé:

[source,bash]
----
mvn spring-boot:run
----

El backend quedarà disponible a:

[source,text]
----
http://localhost:8080
----

== Configuració de l’aplicació mòbil

=== 1. Obrir el projecte

Obrir la carpeta `mobile` amb Android Studio.

=== 2. Configurar la URL base de l’API

Comprovar la configuració de la URL base del backend a la capa de xarxa del projecte mobile.

Per executar en:

* Emulador Android: utilitzar la IP adequada segons configuració local.
* Dispositiu físic: utilitzar la IP local de la màquina on corre el backend.
* Entorn final: actualitzar la `baseUrl` abans de la demo o entrega si cal.

=== 3. Executar l’app

Executar l’aplicació des d’Android Studio en un:

* emulador Android, o
* dispositiu físic Android

== Gestió del projecte amb GitLab

El desenvolupament del projecte s’ha realitzat utilitzant Git i GitLab, seguint una organització basada en branques, issues i merge requests.

=== Branques principals

* `main`: versió estable i lliurable del projecte

=== Branques de treball

* `feature/nom_funcionalitat`
* `bugfix/nom_error`
* `hotfix/nom_emergencia`
* `chore/nom_tasca`
* `experiment/nom_experiment`

=== Flux de treball

* Cada funcionalitat o correcció s’ha desenvolupat en una branca específica.
* Els canvis s’han integrat mitjançant Merge Requests cap a `develop`.
* Les branques fusionades s’han eliminat després del merge.
* La branca `main` conté la versió final per a la demo i el lliurament.
